# ToDo — простой список задач

[Демонстрация на GitHub Pages](https://lesanepc.github.io/ToDo/)

## О проекте
**ToDo** — минималистичное приложение для управления задачами, реализованное на чистом HTML, CSS и JavaScript. Проект создан как лёгкий и понятный пример того, как организовать интерфейс списка задач, фильтрацию по состоянию и приоритетам, и может служить основой для дальнейшего расширения.

## Возможности
- Добавление новых задач
- Фильтрация: все / активные / выполненные
- Сортировка по приоритетам 
- Чистый, минималистичный UI
- Быстрая развёртка через GitHub Pages (страница уже опубликована)

## Технологии
- HTML5
- CSS3
- Vanilla JavaScript (ES6+)
- GitHub Pages — для хостинга

## Как пользоваться
1. Введите текст задачи в поле ввода и нажмите Enter / кнопку добавления.
2. Используйте фильтры (Все / Активные / Выполненные) чтобы отфильтровать задачи.
3. Отмечайте задачи как выполненные/не выполненные.
4. Для удаления/редактирования задач — воспользуйтесь соответствующими кнопками UI.

## Планы для улучшения

### Сохранение данных
- **localStorage**: сохранить массив задач в `localStorage` и загружать его при старте страницы.

### UX / функциональность
- Возможность редактировать задачу (inline-редактирование).
- Кнопка удаления и подтверждение удаления.
- Перетаскивание задач (drag & drop) для реордеринга.
- Поддержка дат/напоминаний и меток.
- Анимации при добавлении/удалении для плавности интерфейса.

### Код и архитектура
- Разделить логику: модуль для работы с данными (Model), модуль для рендера (View) и контроллер (Controller) — MV* подход.
- Вынести JS в отдельный файл (`js/app.js`) и организовать функции: `addTodo`, `toggleTodo`, `deleteTodo`, `render`.
- Добавить валидацию и экранирование пользовательского ввода, чтобы избежать вставки нежелательного HTML.

### Тестирование и CI
- Добавить простые unit-тесты (например, с Jest) для функций работы со списком задач.
- Настроить GitHub Actions для автоматической проверки (lint, тесты).
